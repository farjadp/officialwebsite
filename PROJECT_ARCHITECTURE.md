# معماری و ساختار کلی پروژه (Project Architecture & Overview)

این سند، مرجع اصلی برای درک ساختار، ویژگی‌ها، معماری بک‌اند و مسیر توسعه‌ی وب‌سایت شخصی و پلتفرم محتوایی **Farjad P.** است. این فایل به عنوان یک نمای کلی برای توسعه‌دهندگان، معماران سیستم و همکاران آینده تهیه شده است.

---

## ۱. ساختار کلی پروژه (Project Structure)

این پروژه بر پایه فریم‌ورک قدرتمند **[Next.js 16](https://nextjs.org/)** (با معماری App Router) و زبان **TypeScript** ساخته شده است. تمرکز اصلی روی پرفورمنس بالا، سئوی قدرتمند (SEO) و یکپارچگی ابزارهای هوش مصنوعی با شبکه‌های اجتماعی است.

- **فرانت‌اند (Frontend):** استفاده از React.js به همراه Server Components برای رندر سریع صفحات. استایل‌دهی با **Tailwind CSS** و کامپوننت‌های دسترسی‌پذیر **Radix UI / Shadcn** انجام شده است.
- **بک‌اند (Backend):** بک‌اند به‌صورت یکپارچه در قالب API Routes و Server Actions درون خود Next.js تعبیه شده است.
- **دیتابیس (Database):** پایگاه داده رابطه‌ای **PostgreSQL** که از طریق **Prisma ORM** برای پرس‌وجوهای (Queries) امن و Type-Safe مدیریت می‌شود.
- **زیرساخت و دیپلوی (Infrastructure):** کانتینرایز شده توسط **Docker** و میزبانی‌شده روی **Google Cloud Run** با معماری Serverless.

---

## ۲. لیست کامل ویژگی‌ها (Core Features)

این پلتفرم تنها یک وب‌سایت شخصی نیست، بلکه یک "موتور تولید و توزیع محتوا" است:

* **سیستم مدیریت محتوای پیشرفته (Advanced Blog/CMS):**
  پشتیبانی کامل از نوشتن مقالات با ویرایشگر متن‌باز Tiptap، پشتیبانی از Markdown، مدیریت دسته‌بندی‌ها (Categories)، برچسب‌ها (Tags) و سیستم سری مقالات متصل به هم (Series).

* **آبشار محتوایی هوش مصنوعی (AI Content Waterfall):**
  هسته اصلی تولید محتوای پروژه. این سیستم می‌تواند مقالات نوشته شده را دریافت کرده، "هسته معنایی" (Core Truth) آن‌ها را توسط OpenAI استخراج کند و سپس لحن و متن مناسب برای شبکه‌های اجتماعی مختلف را به‌صورت خودکار بازنویسی کند.

* **توزیع‌گر خودکار شبکه‌های اجتماعی (Social Auto-Publisher):**
  قابلیت انتشار مستقیم و خودکار محتوای تولیدشده توسط هوش مصنوعی در:
  - **کانال‌های Telegram** (با پشتیبانی از فرمت‌بندی).
  - **X / Twitter** (پشتیبانی از اکانت‌های پرمیوم برای متن‌های طولانی یا Thread).
  - **LinkedIn** (اتصال امن از طریق OAuth 2.0 و انتشار مستقیم در پروفایل).

* **داشبورد مدیریت و گزارش‌گیری (Admin Dashboard & Publish Reports):**
  یک پنل کاربری امن برای مدیریت مقالات، آپلود فایل‌های رسانه‌ای، بررسی وضعیت انتشار پست‌ها توییتر/لینکدین/تلگرام، و همچنین نمایش لحظه‌ای خطاهای احتمالی (Logs).

* **آمارگیری زنده و یکپارچه (Live Analytics Analytics):**
  ادغام کامل با سیستم **PostHog**. ردیابی رویدادهای سمت کاربر و سرور که به‌صورت یکپارچه و به شکل یک ویدجت درون خود پنل ادمین (Admin) سایت قابل مشاهده است.

## ۳. معماری سیستم بک‌اند (Backend Architecture)

بک‌اند سایت مبتنی بر یک ساختار **Severless** طراحی شده که می‌تواند ترافیک بالا را مدیریت کرده و هزینه‌های نگهداری را کاهش دهد:

- **مدیریت نشست و احراز هویت (Auth):** از کتابخانه `NextAuth.js` (ورژن 5) استفاده می‌شود. مدیریت ورود ادمین از طریق Credentials و اتصال اکانت‌های شبکه‌های اجتماعی (مثل لینکدین) با پروتکل استاندارد OAuth 2.0 انجام می‌گیرد.
- **تعامل با دیتابیس (ORM):** مدل‌های داده‌ای شامل `User`, `Post`, `Category`, `Tag`, `SocialPublishLog` در فایل `schema.prisma` تعریف شده‌اند. Prisma کلاینت وظیفه ارتباط امن با دیتابیس PostgreSQL که روی سرور خارجی قرار دارد را بر عهده دارد.
- **مدیریت فایل و رسانه (Storage):** تمامی عکس‌ها و ویدیوهای آپلود شده از طریق پنل ادمین، مستقیماً به **Google Cloud Storage (GCS)** ارسال و از طریق CDN سرو می‌شوند تا سرعت لود صفحات سایت افت نکند.
- **پردازش‌های ناهمگام (Async AI & Publishing):** روت‌های `/api/ai/*` و `/api/auth/*` به‌گونه‌ای طراحی شده‌اند که پردازش‌های طولانی‌مدت هوش مصنوعی یا ارتباط با API شبکه‌های اجتماعی به‌صورت موازی انجام شده و نتیجه آن در جدول `SocialPublishLog` برای رهگیری‌های بعدی ثبت شود.

## ۴. ابزارها و سرویس‌های شخص ثالث (Third-Party Integrations)

موفقیت این پلتفرم روی دوش ابزارها و API های قدرتمندی است که در آن ادغام شده‌اند:

1. **OpenAI API (GPT-4 / GPT-4o):** برای خواندن، خلاصه‌سازی و تولید محتوای متناسب با پلتفرم‌های اجتماعی.
2. **Google Cloud Platform (GCP):** برای میزبانی داکر روی *Cloud Run* و ذخیره‌سازی رسانه‌ها روی *Cloud Storage*.
3. **PostgreSQL / Neon (یا ابزارهای مشابه):** دیتابیس اصلی پروژه.
4. **PostHog:** برای Product Analytics، مانیتورینگ ترافیک سایت و بررسی رفتار کاربران.
5. **Twitter API v2:** برای توئیت کردن خودکار.
6. **LinkedIn UGC Posts API:** برای انتشار پست متنی در پروفایل شخصی.
7. **Telegram Bot API:** ارسال محتوا در کانال‌های ارتباطی.
8. **Tiptap:** به عنوان هسته ویرایشگر Rich Text داخل پنل.

## ۵. نقشه راه و توسعه‌های آینده (Future Roadmap & Development)

توسعه‌ی این پلتفرم متوقف نمی‌شود. برای نسخه‌های آینده، پیاده‌سازی ویژگی‌های زیر برنامه‌ریزی شده است:

* **حلقه بازخورد هوشمند (Smart Feedback Loop):**
  اتصال داده‌های تحلیلی PostHog و میزان لایک/بازخورد پست‌ها در شبکه‌های اجتماعی به موتور OpenAI. به این ترتیب هوش مصنوعی یاد می‌گیرد کدام لحن یا ساختار پست بیشترین تعامل (Engagement) را داشته و در آینده پست‌های بهتری می‌نویسد (Self-Optimizing Agent).

* **خبرنامه خودکار (Automated Newsletter):**
  اضافه شدن مدلی برای جمع‌آوری ایمیل‌ها و ارسال خودکار مقالات هفتگی با استفاده از سرویس‌هایی نظیر Resend یا Mailgun.

* **توسعه اتوماسیون شبکه‌های اجتماعی:**
  اضافه شدن پشتیبانی فرمت‌های جذاب‌تر (مانند ساخت اتوماتیک ویدیوهای کوتاه از مقالات متنی، اتصال به Instagram، و پابلیش کراس-پلتفرم روی Medium یا Dev.to).

* **سئوی مبتنی بر هوش مصنوعی (AI-Driven SEO):**
  تولید خودکار تگ‌های Schema، ایجاد ساختار پیوندهای داخلی (Internal Linking) بین مقالات بر اساس شباهت معنایی، و تحلیل و مانیتورینگ جایگاه کلمات کلیدی.

* **بومی‌سازی خودکار محتوا (Auto-Localization):**
  ترجمه خودکار مقالات فنی به زبان‌های دیگر و ایجاد روتینگ چندزبانه (i18n) در Next.js برای جذب مخاطب بین‌المللی بیشتر.
